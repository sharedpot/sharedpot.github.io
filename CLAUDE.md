## Project Overview

**SharedPot** — a static, dependency-free directory of two kinds of resources, plotted on the same map:

1. **Cooking circles** — hand-curated; small groups of neighbours who cook large batches for each other and coordinate via a Telegram/WhatsApp/Signal group.
2. **Food pantries / free food** — auto-imported from public RSS feeds (currently foodpantries.org); shown as a distinct, toggleable category.

Visitors browse, search, use "find near me" to sort by distance, toggle categories on/off (preference persists in localStorage), and click through to the group's invite link or the source's listing page. No backend, no accounts, no build step. Hosted on GitHub Pages.

## File layout

- `index.html` — landing page: header, category-filter chips, search/locate controls, Leaflet map, list.
- `about.html` — why and how to run/join a cooking circle (one page, three sections).
- `submit.html` — embedded Google Form for community submissions.
- `app.js` — Leaflet, multi-source fetch, search filter, geolocation + Haversine, category toggles.
- `styles.css` — layout grid, platform badges, category colors and chips.
- `groups.json` — hand-curated cooking circles.
- `food_resources.json` — generated by `scripts/fetch_resources.py`. Never edit by hand.
- `scripts/fetch_resources.py` — pulls food-aid RSS, geocodes via Nominatim, writes JSON.

## Entry schema

```json
{
  "id": "kebab-case-unique",
  "category": "circle | pantry | free-food",
  "name": "Display name",
  "description": "Short blurb.",
  "platform": "telegram | whatsapp | signal",   // circles only
  "url": "https://...",                         // chat invite (circles) or source listing
  "lat": 00.0000,
  "lng": 00.0000,
  "address": "Area, City, Country",
  "source": { "name": "...", "url": "..." }     // non-circle entries; attribution
}
```

Conventions:
- `lat` / `lng` are pre-computed, 4–5 decimals.
- `id` is unique, lowercase, kebab-case. Imported entries use a source prefix (e.g. `fp-...` for foodpantries.org).
- `platform` only applies to `circle` entries; other categories ignore it.
- Entries with unknown `category` or non-finite coordinates are filtered out at load time.
- Keep both JSON files valid (no trailing commas, no comments).

## Categories and colors

- `circle` → coral (`--cat-circle: #E66B3D`)
- `pantry` → sky blue (`--cat-pantry: #0EA5E9`)
- `free-food` → leaf green (`--cat-free-food: #16A34A`) — reserved for v2

The third chip is added to `index.html` only when `food_resources.json` contains entries with `category: "free-food"`.

## Refresh food-aid data

```bash
uv run scripts/fetch_resources.py
git add food_resources.json
git commit -m "data: refresh food resources"
git push
```

The script:
- Fetches foodpantries.org RSS, parses with `feedparser`.
- Extracts the US-format street address from each item's description (or fetches the detail page once if the summary doesn't carry one).
- Geocodes via `nominatim.openstreetmap.org` at 1 req/sec (their policy).
- Caches HTTP fetches in `scripts/.fetch_cache.json` and geocoding in `scripts/.geocode_cache.json` (both gitignored).
- Idempotent: rerunning merges by `id` and produces the same JSON if nothing changed upstream.

To add a new source (e.g. freefood.org once they reply with a feed), implement a class with `fetch_items()` and `parse_entry()` matching the `FoodPantriesOrg` shape and add it to `sources` in `main()`.

## Submissions flow

1. Visitor fills the Google Form embedded on `submit.html`.
2. Responses land in a Google Sheet.
3. Owner reviews, geocodes the address, appends to `groups.json` (with `category: "circle"`), commits, pushes.

## Run locally

```bash
python3 -m http.server 8000
```

Open <http://localhost:8000>. (`file://` won't work — geolocation and `fetch()` need an HTTP origin.)

## Manual test checklist

- Map loads with markers in the right colors per category; clicking a marker shows a popup with badge + link.
- Typing in the search box filters list and markers.
- Toggling a category chip hides/shows that category in both list and map; reload preserves the choice.
- "Find near me" prompts for permission, drops a marker at the user's location, sorts the list by distance.
- Radius selector hides entries beyond the chosen distance.
- Cooking-circle CTAs say "Open chat ↗"; pantry/free-food CTAs say "Visit listing ↗" with `via FoodPantries.org` (or whichever source) attribution.
- Clicking a list item pans the map and opens the popup.
- Header `About` link → `about.html` renders.

## Deploy

GitHub Pages: **Settings → Pages → Source: `main` / root**. Every push to `main` redeploys. No build step, no Actions workflow needed.
